**Deno 後端開發、Router 路由管理、與 SQLite 資料庫實作** 的教學過程。

### 一、 技術重點整理

#### 1. Router (路由器) 的概念

* **功能**：比簡單的 `if/else` 更系統化地管理網址路徑。
* **軟體路由器 vs 硬體路由器**：硬體路由器處理 TCP/IP 封包；軟體路由器（如 Oak 的 Router）處理網址（URL）的路由。
* **參數路由 (Parametric Routing)**：例如 `/book/:id`，網址中的 `:id` 會被當作參數抓取，方便顯示特定編號的資料。

#### 2. 資料迭代器 (Iterator) 與 陣列 (Array)

* `map.values()` 回傳的是 **Iterator (循環器/迭代器)**，它不是真正的陣列。
* 如果要將它變成陣列以便後端回傳（如 JSON 格式），需要使用 `Array.from()` 進行轉換。

#### 3. 留言板系統 (Blog System) 運作流程

* **List (列表)**：顯示所有貼文。
* **New (新增頁面)**：顯示表單讓使用者填寫標題與內容。
* **Create (創建)**：
* 接收由表單傳來的 **POST** 請求。
* 讀取表單資料（使用 `body.form()`）。
* 將資料存入陣列（暫時性）或資料庫。
* **重新導向 (Redirect)**：建立完成後自動跳回根目錄 `/` 刷新畫面。


* **Show (顯示單筆)**：根據 ID 從資料庫/陣列抓取特定內容，並使用 `<pre>` 標籤保留文字換行格式。

#### 4. SQLite 資料庫實作

* **特性**：輕量、無需啟動伺服器服務（非 Server-based），適合個人專案或教學使用。
* **SQL 基礎指令**：
* `CREATE TABLE IF NOT EXISTS`：建立表格。
* `INSERT INTO`：新增資料。
* `SELECT`：選取並查詢資料。


* **自動化欄位**：
* `PRIMARY KEY AUTOINCREMENT`：自動生成的唯一代號（ID）。
* `DEFAULT CURRENT_TIMESTAMP`：自動記錄資料存入的時間。

本週共有兩個實作練習，請參考老師提供的範例代碼進行修改。

#### **作業六：SQLite 基礎操作 (SQLite Hello)**

* **目標**：修改 `sqliteHello.ts` 範例，建立一個貼文表格。
* **要求**：
1. 表格欄位需包含：`id` (主鍵)、`time` (時間戳記)、`title` (標題)、`body` (內容)。
2. 程式執行時需**手動新增三筆貼文**進資料庫。
3. 最後將資料庫內容（包含自動生成的 ID 與時間）讀取並印出在終端機。



#### **作業七：具有時間戳記的留言板 (Blog with Timestamp)**

* **目標**：將上週的留言板系統升級，加入時間顯示功能。
* **要求**：
1. 修改 `Render` 模組，在顯示列表（List）與單筆（Show）時，需額外顯示貼文的**日期與時間**。
2. （進階）將原本存在記憶體（陣列）的資料，改為使用 **SQLite** 進行持久化存儲（關機後資料不會消失）。
